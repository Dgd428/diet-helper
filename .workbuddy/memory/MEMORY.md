# 饮食助手 - 项目记忆

## 产品定位
减肥人群 + 家庭成员的饮食规划工具，降低每日"吃什么"的决策成本。

## 核心设计原则
- **温暖不爹味**：不催促、不 judge，像朋友一样陪伴
- **傻瓜式**：步骤详细到火候和时间，纯小白友好
- **多成员**：核心差异化特征，支持不同饮食目标的人在同一首页

## 关键功能
- 按时段切换首页（早餐/午餐/晚餐/禁食 四个时段）
- 多成员卡片并列显示
- 锁定→自动热量平衡→减脂餐7天不重复
- 周五采购清单气泡（合并重复食材）
- 标记已吃（可选、可补标记）
- 食材库（保存方法+选购+关联菜品）

## 技术栈
- 当前阶段：纯静态 HTML + JSON 数据文件
- 未来考虑：微信小程序

## 数据架构（v2.0 拆菜重构）
- `data/dishes.json`：108道独立菜品，按 类型_餐别_品类 组织（如 减脂_午餐_主菜）
- `data/rules.json`：组合规则（每餐的热量范围 + 必选/可选品类）
- `data/ingredients.json`：93种食材百科
- `data/recipes.json`：旧版64道套餐（保留参考，已废弃不用）
- 组合引擎：每餐从各品类随机抽1道 → 热量达标输出 → 用户可逐道替换

## 组合规则速查
| 餐别 | 减脂组合 | 减脂热量 | 营养热量 |
|------|---------|:------:|:------:|
| 早餐 | 1主食+1蛋白(+饮品/蔬果可选) | 280-380 | 380-580 |
| 午餐 | 1主菜+1配菜+1主食 | 380-520 | 550-750 |
| 晚餐 | 1主菜(+1配菜) | 200-350 | 450-650 |
